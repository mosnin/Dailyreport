# UI Architecture

**Last updated:** May 2, 2026
**Version:** 1.1

---

## Stack

- **Framework:** Next.js 14 App Router (`app/` directory, all routes under `app/(dashboard)/`)
- **Styling:** Tailwind v4 + shadcn/ui v4 primitives (no `asChild` — use plain `<Link>` instead of `<Button asChild>`)
- **Animation:** `motion/react` (`AnimatePresence`, `motion.div`, `fadeUp` helper from `lib/motion.ts`)
- **Icons:** `lucide-react`
- **State / data:** Convex real-time (`useQuery`, `useMutation` from `convex/react`)
- **Auth:** Clerk (`useUser`, `useClerk`, `useConvexUser` hook)

---

## Application Shell

All authenticated routes live under `app/(dashboard)/layout.tsx`, which renders:

```
┌─────────────────────────────────────────────────────┐
│  Sidebar (desktop ≥ lg)          │  Page content     │
│  BottomTabBar (mobile < lg)      │                   │
└─────────────────────────────────────────────────────┘
```

### `components/layout/Sidebar.tsx`
- Collapsible (persisted in `localStorage` as `"sidebar-collapsed"`)
- Collapses to icon-only at 60px width; expands to 224px
- Contains: logo, today-progress strip (3 dots for report/affirmations/viz done), quick search button, voice agent button, nav sections, user info, settings, sign-out
- Active item animated with `motion` `layoutId="nav-active-bg"` shared layout

### `components/layout/BottomTabBar.tsx`
- Shown only on `< lg` screens (`lg:hidden`)
- 5 primary tabs: Today, Daily, Affirmations, Goals, More
- "More" opens a `Sheet` drawer (shadcn) with the full nav tree
- Voice agent button is a center pill (`open-voice-agent` custom event)
- Safe-area inset for iPhone notch via `env(safe-area-inset-bottom)`

## Key Pages in Detail

### `/today` — Command Center
The primary screen users open every morning. Data sources:
- `api.agentJobs.listRecentJobs` — filtered client-side for `createdAt >= startOfToday()`
- `api.externalTasks.getTasksByUser` — open tasks from all connected platforms
- `api.goals.getCurrentSummary` — completion rates by period
- `api.integrations.getUserIntegrations` — which platforms are connected

Sections rendered:
1. **Date + greeting** — time-of-day aware ("Good morning/afternoon/evening, Alex.")
2. **Morning briefing card** — only shown when a done job with `result.briefing` and `intent.startsWith("Morning briefing")` exists for today. Styled with `bg-primary/[0.04]` border tint to distinguish from regular cards.
3. **Open tasks** — sorted overdue → due today → upcoming → no date. Overdue = red `AlertTriangle`. Cap at 20 shown; remainder gets a "ask your assistant" nudge.
4. **Goals needing attention** — only shown when any period has `completed < total`.
5. **Conversation thread** — all user-initiated jobs for today, rendered as chat bubbles. User message = right-aligned primary bg. Agent response = left-aligned with `Bot` icon avatar.
6. **Agent command bar** — textarea (`Enter` sends, `Shift+Enter` newlines). Shows connected platform count. Fires `createJob` mutation then `POST /api/agent/trigger`.

### `/agent` — Agent Page
The legacy/power-user agent view. Shows:
- Latest done briefing from recent jobs
- Live active job status panel (progress log, status badge)
- Task feed (`externalTasks`)
- Agent command form
- Collapsible recent job history

## Data Flow Pattern

All data is Convex real-time — no manual `fetch` calls for application data:

```typescript
// Reading data (live subscription — re-renders on any backend change)
const tasks = useQuery(api.externalTasks.getTasksByUser, { userId: convexUserId });

// Writing data
const createJob = useMutation(api.agentJobs.createJob);
await createJob({ userId: convexUserId, intent: "..." });
```

Pages use `"skip"` as the query arg when `convexUserId` is not yet loaded:
```typescript
useQuery(api.goals.getCurrentSummary, convexUserId ? { userId: convexUserId } : "skip")
```

### `useConvexUser` hook (`hooks/useConvexUser.ts`)
Returns `{ convexUserId, convexUser, isLoading }`. On first mount, calls `api.users.getOrCreate` with the Clerk identity to upsert the Convex user record. Every page that needs the Convex user ID uses this hook.

## Animation System

All page content uses staggered `fadeUp` animations from `lib/motion.ts`:

```typescript
import { fadeUp } from "@/lib/motion";
<motion.div {...fadeUp(0)}>first section</motion.div>
<motion.div {...fadeUp(1)}>second section</motion.div>
```

`fadeUp(index)` returns `{ initial, animate, transition }` props with a staggered delay based on index. `AnimatePresence` is used for conditional sections (e.g. morning briefing card appearing after the job completes).

## Auth Integration

All dashboard pages are protected by `middleware.ts` (`clerkMiddleware`). Public routes: `/`, `/sign-in(.*)`, `/sign-up(.*)`.

The Clerk JWT is passed to Convex via `ConvexWithClerkProvider` (`components/ConvexWithClerkProvider.tsx`), which wraps the app with `ConvexProviderWithClerk`. Convex validates the JWT against the Clerk JWKS endpoint configured in `convex/auth.config.ts`.

Server-side auth (API routes):
```typescript
import { auth } from "@clerk/nextjs/server";
const { userId } = await auth(); // Clerk userId (clerkId), not Convex ID
```

## Notification / Status Dots

The sidebar today-progress strip and mobile tab dots come from `useTodayStatus(convexUserId)` (`hooks/useTodayStatus.ts`). It queries whether the user has submitted their daily report, done affirmations, and completed their visualizations for today.

- Green dot = complete
- Amber dot = not complete
- No dot = not applicable


## Recent UX Updates (May 2, 2026)

### `/dashboard` auto-sync on app open
- On dashboard load, when a user has connected integrations, the client now auto-creates an agent job and triggers `/api/agent/trigger` to pull latest external updates/tasks.
- The dashboard renders a lightweight "Live from connected apps" snapshot showing open synced task count once data arrives.

### `/chat` agentic experience
- Chat now uses a richer "Thinking" animation chip instead of a minimal line pulse.
- Assistant replies are rendered with progressive streaming-style reveal for a more real-time agent feel on both web and PWA surfaces.
